Práctica 3: Sistema de usuarios con validaciones y excepciones
Se desarrolló un sistema orientado a objetos en PHP que implementa herencia, validación de datos y manejo de excepciones.
Usuario: clase base con validación de correo.
Admin: hereda de Usuario y define el rol de administrador.
Alumno: hereda de Usuario y agrega el atributo matrícula.

Se implementaron bloques try/catch para capturar excepciones cuando el correo no es válido.
Iniciar Apache en XAMPP.
Abrir en el navegador:
http://localhost/desarrollo-web-avanzado-fimaz-uas/parcial-1-poo/practica-3/

Se mostrarán usuarios válidos y errores controlados.
